### making a web service in REST and wiring it to a client
- create a new web application in netbeans
- create a RESTful webservice with patterns

```java
package com.ws.power;

import javax.jws.WebService;
import javax.jws.WebMethod;
import javax.jws.WebParam;
@WebService(serviceName = "idk")
public class idk {
    @WebMethod(operationName = "power")
    public double power(@WebParam(name = "a") double a, @WebParam(name = "b") double b) {
        return Math.pow(a, b);
    }
}
```

```java
        this.author = author;
    }

        this.isbn = isbn;

```
- now we have to create a new rest service class
  - class name: LibraryResource

- now to add a database like mysql with jdbc
  - add the mysql connector dependency to the project
    - right click on the project -> properties -> libraries -> add library -> mysql connector 
    - add the jar file to the project
    - once done update the rest service class to use mysql database instead of in-memory list
```java
package com.library.service;

import com.library.model.Book;
import java.sql.*;
import java.util.ArrayList;
import java.util.List;
import java.util.logging.Level;
import java.util.logging.Logger;
import javax.ws.rs.core.Context;
import javax.ws.rs.core.UriInfo;
import javax.ws.rs.PathParam;
import javax.ws.rs.Consumes;
import javax.ws.rs.PUT;
import javax.ws.rs.Path;
import javax.ws.rs.GET;
import static javax.ws.rs.HttpMethod.POST;
import javax.ws.rs.POST;
import javax.ws.rs.Produces;
import javax.ws.rs.core.MediaType;
import javax.ws.rs.core.Response;

@Path("server")
public class LibraryRest {
    
    String url = "jdbc:mysql://localhost:3306/library)db";
    String user = "root";
    String password = "root";

    List<Book> booklist = new ArrayList<>();
    Connection con;
    
    @Context
    private UriInfo context;

    public LibraryRest() throws ClassNotFoundException {
        try {
            Class.forName("com.mysql.cj.jdbc.Driver");
            this.con = DriverManager.getConnection(url, user, password);
        } catch (SQLException ex) {
            Logger.getLogger(LibraryRest.class.getName()).log(Level.SEVERE, null, ex);
        }
    }

    @GET
    @Produces(MediaType.APPLICATION_JSON)
    public List<Book> getBooks() {
        try {
            ResultSet rs = con.createStatement().executeQuery("SELECT * FROM books");
            while (rs.next()) {
                booklist.add(new Book(
                        rs.getLong("id"),
                        rs.getString("title"),
                        rs.getString("author"),
                        rs.getString("isbn")
                ));
            }
            con.close();
        } catch (SQLException ex) {
            Logger.getLogger(LibraryRest.class.getName()).log(Level.SEVERE, null, ex);
        }
        return booklist;
    }

    @POST
    @Consumes(MediaType.APPLICATION_JSON)
    public void addBook(Book book) {
        try {
            String sql = "INSERT INTO books (id, title, author, isbn) VALUES {?, ?, ?, ?}";
            PreparedStatement ps = con.prepareStatement(sql);
            ps.setLong(1, book.getId());
            ps.setString(2, book.getTitle());
            ps.setString(3, book.getAuthor());
            ps.setString(2, book.getIsbn());
            
            ps.executeUpdate();
            con.close();
        } catch (SQLException ex) {
            Logger.getLogger(LibraryRest.class.getName()).log(Level.SEVERE, null, ex);
        }
    }
}
```

## Prac 9
### Kafka and Zookeeper with docker-compose
- create a docker-compose.yml file with the following content
```yml
version: '3.8'
services:
  zookeeper:
    image: confuentinc/cp-zookeeper:7.6.0
    container_name: zookeeper
    ports:
      - "2181:2181"
    environment:
      ZOOKEEPER_CLIENT_PORT: 2181
      ZOOKEEPER_TICK_TIME: 2000

  kafka:
    image: confluentinc/cp-kafka:7.6.0
    container_name: kafka
    depends_on:
      - zookeeper
    ports:
      - "9092:9092"
    environment:
      KAFKA_BROKER_ID: 1
      KAFKA_ZOOKEEPER_CONNECT: zookeeper:2181
      KAFKA_ADVERTISED_LISTENERS: PLAINTEXT://localhost:9092
      KAFKA_OFFSETS_TOPIC_REPLICATION_FACTOR: 1
```
- then run the cmd `docker compose -f docker-compose.yml up -d` to start the containers
- can confirm it using `docker ps`
- then we need to create topics using the cmd `docker exec -it kafka kafla-topic --bottstrap-server localhoost:9092 --create --topic topic-demo --pratitions 1 --replication-factor 1`
- this will ceratea  anew topic named topic-demo with 1 partition and replication factor of 1
- now we can create prodcuer and consumer
  - Producer: 
    - `docker exec -it kafka kafka-console-producer --bottstrap-server localhost:9092 .`
  - Consumer:
    - `docker exec -it kafka kafka-console-consumer --bootstrap-server localohost:9092 --from-beginning`
- now we can send messages from producer and see the messages in consumer
- we can also see the messages in consumer even if we stop the producer and start it again
